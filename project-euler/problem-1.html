Multiples of 3 and 5

Problem #1 of Project Euler series

Question

If we list all the natural numbers below 10 that are multiples of 3 or 5, we get 3, 5, 6 and 9. The sum of these multiples is 23.

Find the sum of all the multiples of 3 or 5 below 1000.

One-liner solution


def oneliner():
    print(sum([i if i % mult_one == 0 or i % mult_two == 0 else 0 for i in range(1, lim_up)]))

lim_up, later assigned the upper limit of 1000, is used to create an array of integers from 1 to 1000.
We loop through this array checking at each stage whether the integer is divisible by mult_one or mult_two.
If it is divisible, we retain the value, otherwise we substitute it by a zero to negate its effect.
Finally we sum the resulting array to satisfy the question requirements and print its output.

Throughout the series I occasionally include one-liner solutions.
One-liner solutions simply look cool and mostly do not offer any operational advantages.
However, some claim they can be easier to read and interpret by humans/users.

Brute-force solution


def brute_force():
    sum_mults = 0

    for i in range(1, lim_up):
        if i % mult_one == 0 or i % mult_two == 0: sum_mults = sum_mults + i

    print(sum_mults)

We create the variable sum_mults to store the sum of applicable integers as we go along.
We loop through each integer in our range, starting from 0 to the upper limit of 1000.
We check at each step whether the integer is divisible by 3 or 5 and if so, we add it to our sum.
We print our output to the console.

Throughout the series I occasionally include brute-force solutions.
Brute-force solutions take a simplistic approach, testing or calculating all possible values in a range.
They are not efficient and can use up too many resources for a rather long period of time.
On the other hand, they are inherently simple and straight forward.

Strategic solution


def strategic():
    #1-333 sum *3 + 1-199 sum *5 - 1-66 sum *15
    a = ((1 + int((lim_up - 1) / mult_one)) / 2 * int((lim_up - 1) / mult_one) * mult_one)
    b = ((1 + int((lim_up - 1) / mult_two)) / 2 * int((lim_up - 1) / mult_two) * mult_two)
    c = (1 + int((lim_up - 1) / (mult_one * mult_two))) / 2 * int((lim_up - 1) / (mult_one * mult_two)) * mult_one * mult_two

    print(int(a+b-c))

In the range 1 to 1000, there are 1000 / 3 number of multiples of 3. Same goes for 1000 / 5 multiples of 5.
However, in this approach, we are double counting integers that are the multiples of both 3 and 5, basically the multiples of 15.
So, we sum the numbers from 1 to 333 to see how many multiples of 3 we have, and multiply them by 3 (as they are multiples of 3)
We do the same for 1 to 200 for 5, and 1 to 66 (1000 / 15) for 15.
We sum the result for 3 and 5, and later deduct the result for 15 as these were double counted.
In the code above, a is for the result of 3, b is for the result of 5, and c is for the result of 15.

Throughout the series I occasionally include strategic solutions.
These solutions should be given priority. They are usually the most efficient approach of solving the question that I can think of.
They take advantage of logical deductions implied by the question.
